Background

The U.S. Department of Agriculture Natural Resources Conservation Service (NRCS) defines ecological sites as "a distinctive kind of land with specific physical characteristics that differs from other kinds of land in its ability to produce a distinctive kind and amount of vegetation" (USDA-NRCS). Ecological sites determine not only the ecological potential of a site, but also how a site can respond to natural disturbance and management actions. Ecological sites have been delineated for many areas using physical and biotic features and are linked to soil map units. Soils, climate, hydrology, geology, physiographic features, plant species occurrences, plant community composition, annual biomass production, and plant-wildlife interactions all contribute to the properties of ecological sites. Ecological Site Descriptions (ESDs) describe the underlying factors associated with a specific ecological site, as well as different ecological states within a given site based on factors such as grazing, fire, drought, and other management actions. For an example of an ESD, see EDIT Jornada.

Ecological Sites

Using ecological monitoring data, the overall status of an ecological site can be summarized into an Ecological Site Summary (ESS). At the site level, ecological site summaries generated from monitoring data can tell us about the condition and health of ecological sites based on indicators such as structure and functional group, species dominance, species occurrences, canopy gap measurements, and soil stability measurements. These indicators can help inform the user of the ecological state of a given plot or study site (i.e. shrubland state vs. grassland state). Identifying the ecological state allows users to not only determine the degree of departure from the corresponding reference ecological states, but also to inform management and restoration actions by determining the best pathway to a less degraded state. These shifts between states can be modeled with a State and Transition Model (STM). For more information on state and transition models, see Bestelmeyer et al., 2017.

A major data source for developing these summaries is the Assessment, Inventory, and Monitoring (AIM) Program along with the Landscape Monitoring Framework (LMF). AIM and LMF include almost 25,000 plots throughout the western United States (see AIM Data Portal) stored in the Terrestrial AIM Database (TerrADat). Data garnered from AIM and LMF plots is both generalizable and scalable and thus applicable to management decisions across multiple disciplines at both local and national levels.

The Ecological Indicators Summary (EIS) tool has been developed to help users visualize the overall status of their region of interest within a specific ecological site, or across an ecological site group. This information can be summarized within a given administrative state or across political boundaries for landscape scale analysis. Users can query summary tables, download full or filtered summary tables to view in Excel, explore interactive figures, and download static figures for their own use.

This EIS information can be included in decision making processes for Bureau of Land Management (BLM) managers, as it can help inform land use plans, grazing permit renewals, shrub or invasive species treatments, and/or seed menu development for ecological restoration and reclamation. As new data become available or lines of inquiry shift, the summary tables and report can be easily updated.

Using the EIS Tool

This tool generates many tables and figures that can be queried, filtered, and downloaded to meet your management and reporting needs. Multiple output types are used to meet multiple needs, so select the most relevant and useful output. In cases where ESDs are not developed, grouping AIM data by other features can provide similar information, and potentially lead to the development of new ESDs. For this purpose, it may be informative to group data by features such as biophysical indicators, location, or project. Spatial groupings may also be beneficial if you want to compare data in the same ecological site across multiple management units such as grazing allotments, pastures, prescribed burn areas, etc. The report you generate can be customized to select and compare indicators by both spatial and non-spatial groupings.

Step 1: Access the tool

To use the EIS Tool, navigate to https://www.stone-env.net/esst/.

Click "Get Started" to pull up a map and query the data. Use the tab to expand and collapse the form and map in the application.

Step 2: Select attributes and indicators

Use the attribute buttons at the top of the form to filter the data by "Soil Attributes", "Plant Attributes", "Erosion Indicators", or "Indicators of Rangeland Health". Select properties under one or more of these 4 categories to use in your query. You will be able to sort your data by indicator and set benchmarks in a later step.

The EIS tool is organized by query group, where a query group contains indicators and an area of interest designed to answer a specific ecological question. Click "Add New Row" to start customizing your query group. Use the dropdown menu under "Property" to add one of the indicators that you selected above to your query statement. Choose an operator, then input a value to complete the statement. Add additional rows to your group to complete your query statements. You can come back and edit these statements later too, if need be. If you want to remove a query statement, you can click the yellow "X" button to the right of the row.

Include time period
This optional step queries the data over a specific time period. In the query group, click the "Add New Row" button. Use the new row's "Property" dropdown to select "Date Visited" under "All Indicators". Then select one of the "Operator" dropdown options:

Operator Description
= Query the data for a specific date entered in the "Value" input using mm/dd/yyyy format
!= Query the data for all time EXCEPT a specific date entered in the "Value" input using mm/dd/yyyy format
< Query the data for all dates BEFORE a specific date entered in the "Value" input using mm/dd/yyyy format
≤ Query the data for all dates BEFORE AND INCLUDING a specific date entered in the "Value" input using mm/dd/yyyy format
> Query the data for all dates AFTER a specific date entered in the "Value" input using mm/dd/yyyy format
≥ Query the data for all dates AFTER AND INCLUDING a specific date entered in the "Value" input using mm/dd/yyyy format
>= & <= Query the data for all dates BETWEEN AND INCLUDING two specific dates entered in the inputs using mm/dd/yyyy format
> & < Query the data for all dates BETWEEN two specific dates entered in the inputs using mm/dd/yyyy format
Year Choose a specific year in the "Value" dropdown to query data by year.

Step 3: Select an area of interest (AOI)

Click "Add AOI" for to select the area of interest that you want to include in your query. You may select one or all of the following choices.

Draw Polygon: Use the Zoom In and Zoom Out buttons on the map. After selecting "Add AOI" > "Draw Polygon", click on the map to start drawing. Continue clicking until the polygon is your intended size and shape. You may add additional separate polygons to the same query group. You will be able to manage overlapping polygons and exclude features in a later step.

Project: Use this option to select one or more projects using a Project Key, ESD, MLRA, or Ecoregion Level #3. ESDs and MLRAs are described in more detail in EDIT Jornada. Ecoregions can be found on the EPA's website.

Upload Shapefile: This option allows you to upload a shapefile from your device into the tool. Note: you may combine any of these AOI options in one query. For example, you can select both an MLRA and a shapefile or you can select an ESD, a shapefile, and a hand-drawn polygon and they will be combined into a single area of interest. For problems with the shapefile upload, see "Why is my shapefile not working?" in the Resources section.

Step 4: Manage query group and geodata

If you would like to begin a new query, click the "Add New Group" button to add a query group. Then add a new row with with the "Add New Row" button. Choose a property from the Property dropdown menu. You may also choose an operator and a value. Alternatively, you may add multiple properties to the original query group by clicking the "Add New Row" button.

You may also create a duplicate of your existing query group by selecting "Clone Group". This clone can then be modified and renamed. You may delete groups by using the filter button and selecting "Delete Group." You may delete queries within a group by clicking the X at the end of the line. You may name your groups using the filter button and selecting "Rename Group".

If you wish to manage the geodata, and exclude or select AOIs, click the "Query Group Settings" button and choose "Manage Geodata". Then click the "Select AOIs" button. Use the GeoData Manager to inspect your AOI selections and the number of unique sites they each contain. Click a table row to zoom the map to that selection's extent. By default, unique sites in each selection are added together in the query. To subtract a selection's sites from the query, click "Exclude". You may add it back to the selection by clicking "Select". To remove a selection, click the "Remove AOIs" button. Press the "Cancel" button and then the "Close" button to return to the Query Form.

Step 5: View and interact with query results

Once you have set your query group properties and areas of interest, click "Submit Query" to run the query. You may need to wait a moment for the data to load. If there are errors, follow the instructions provided to fix the error.

Afer the query is done, click the "Results" tab and use the "Graph" and "Table" tabs to view the data.

Step 6: Compare data from multiple query groups

Click the button "Select Results from Query Group" and select the query groups you want to compare then click "Compare". You may also choose to look at a single query group. You can filter by typing a search into the "Filter" bar. You can also click on the column name to sort by a property.

Step 7: Export data

If you want to export the current table, go to the "Table" tab and click the download button and an Excel (.xlsx) file will be generated. You may also change which table you are viewing.

If you want to export a document with the graphs included, go to the "Graph" tab and click the download button and a .pdf file will be generated.

Glossary of Terms and Acronyms

Acronyms

  • AIM: Assessment, Inventory, and Monitoring Program
  • BLM: Bureau of Land Management
  • ESD: Ecological Site Description
  • ESS: Ecological Site Summary
  • LMF: Landscape Modeling Framework
  • MLRA: Major Land Resource Area
  • STM: State and Transition Model
  • TerrADat: Terrestrial AIM Database

Terms

  • Annual biomass production: The amount of organic plant matter produced in a given area over the course of a year.
  • Ecological potential: The greatest potential fractional cover (bare ground, herbaceous, litter, shrub, and sagebrush) represented in the least disturbed areas and most productive years of the record for a given plot of land. It is a measurement of how much biomass could grow in a region given perfect conditions.
  • Ecological restoration and reclamation: The process of assisting the recovery of severely degraded ecosystems to benefit native biota through the establishment of habitats, populations, communities, or ecosystems that are similar, but not necessarily identical to surrounding and naturally occurring ecosystems. (Gerwing et al., 2021).
  • Ecological site: A distinctive kind of land with specific soil and physical characteristics that differ from other kinds of land in its ability to produce distinctive kinds and amounts of vegetation, and in its ability to respond similarly to management actions and natural disturbances (USDA-NRCS).
  • Ecoregion: A major ecosystem defined by distinctive geography and receiving uniform solar radiation and moisture. They are areas where ecosystems (and the type, quality, and quantity of environmental resources) are generally similar. Designed to serve as a spatial framework for the research, assessment, and monitoring of ecosystems and ecosystem components, ecoregions denote areas of similarity in the mosaic of biotic, abiotic, terrestrial, and aquatic ecosystem components with humans being considered as part of the biota. Ecoregions are identified by analyzing the patterns and composition of biotic and abiotic phenomena that affect or reflect differences in ecosystem quality and integrity (Omernik 1987). These phenomena include geology, landforms, soils, vegetation, climate, land use, wildlife, and hydrology. More details can be found on the EPA's website.
  • Management actions: Decisions and actions made under an approach to maintaining the health and productivity of an ecosystem while sustaining appropriate human uses. More details can be found in Brussard et al., 1998.
  • Natural Disturbance: Disturbances are events, like tornados, wildfires or floods that cause marked changes to the impacted area. Examples of ecological disturbances include fires, landslides, flooding, windstorms and insect and pest outbreaks. Disturbances often come in the form of short-term or temporary changes to the landscape but can have very significant ecosystem impacts. These events often act quickly but with great impact and thereby are able to promote changes to the physical structure of the system. For example, a fire burning over a mature forest results in a different composition of species after the disturbance event. Other types of disturbances may occur over longer, persistent timeframes that also can promote changes to the biodiversity or health of the system. More details can be found on the USDA's website.
  • Physiographic features: The physical features and processes of landforms and their relation to geologic structures, including the soils, topography, land slope and aspect, and local climate, that influence the form and species composition of plant communities.
  • Plant community composition: A description of vegetation in a fixed area by the species present and their abundance. (Jaroszynska et al.).
  • Plant-wildlife interactions: Relationships between the plants and animals in a given ecosystem. These relationships include predation, parasitism, commensalism, or mutualism and do not require direct physical contact. Important plant-wildlife interactions are pollination, seed dispersal, and herbivory. The plants and animals have co-evolved these adaptive mechanisms to maximize the chance of survival.
  • State-and-Transition model: These models synthesize literature and informal knowledge tied to particular ecological sites to distinguish changes in vegetation and soils that are easily reversible versus changes that are subject to thresholds beyond which reversal is costly or impossible. The models describe all possible states, community phases (i.e., easily-reversible variants of states), and transitions between communities and states. Transitions contain information about mechanisms, triggers, thresholds, and indicators of threshold development. More details can be found on the Jornada website.

Why is my shapefile not working?

Although a very common format for storing spatial vector information (polygons, lines, points), shapefiles can contain geometry errors or other misinformation that prevents them from functioning properly. If you are using a zipped shapefile and are not returned results, please consider the following:

  • The shapefile must be saved as a .zip file and contain the .shp, .shx, and .prj files of the shapefile. Only include one shapefile per .zip file.
  • Shapefiles may not work with all web browsers. Please use Google Chrome for best results.
  • 3D shapefiles are not supported, please convert to 2D.
  • Shapefiles can contain geometry errors. Please check and fix the geometries of the shapefile and try again.
  • Shapefiles with complex geometries (i.e., polygons with many vertices) or many features will result in an error. Please reduce the number of features or simplify them.
  • Sliver polygons (really, really small polygons that are likely mistakes) will result in an error. Please check and fix the geometries of the shapefile.
  • Although multipart polygons are compatible with the EIST, singlepart polygons are recommended. Convert to singlepart polygons.
  • Only the first 50 features of the shapefile will be loaded; the remaining features will be discarded.
